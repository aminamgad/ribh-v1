# قائمة مهام تحسين UI/UX لعرض معلومات Webhook

حالياً يتم عرض معلومات Webhook عبر `window.alert()` الأصلي للمتصفح، مما يعطي تجربة مستخدم بسيطة وغير متناسقة مع بقية التطبيق.

---

## المشاكل الحالية مع `alert()`

- [ ] **مظهر خارجي:** `alert()` يبدو كنافذة منفصلة عن المتصفح وليست جزءاً من التطبيق
- [ ] **عدم اتساق التصميم:** لا يتبع ثيم التطبيق (داكن/فاتح) ولا الخطوط والألوان المخصصة
- [ ] **عدم القدرة على نسخ النص:** المستخدم لا يستطيع نسخ Webhook URL أو Store ID بسهولة
- [ ] **عرض غير منظم:** النص الطويل (خاصة URLs) قد يظهر بشكل غير مرتب
- [ ] **لا يوجد تحسين للـ RTL:** `alert()` قد لا يعرض النص العربي بشكل مثالي في بعض المتصفحات
- [ ] **زر واحد فقط:** لا توجد أزرار إضافية (مثل نسخ الرابط، فتح التكاملات، إلخ)

---

## التعديلات المقترحة

### 1. استبدال `alert()` بمكون Modal مخصص

- [ ] **إنشاء مكون `WebhookInfoModal`** أو إعادة استخدام/توسيع `ConfirmationModal` أو مكون مشابه
- [ ] **تصميم يتوافق مع التطبيق:** نفس الـ cards، الألوان، الخطوط، الداكن/الفاتح
- [ ] **واجهة واضحة:** عنوان "معلومات Webhook"، وعرض البيانات في صفوف منظمة (label + value)

### 2. عرض البيانات بشكل منظم

- [ ] **استخدام صفوف/قوائم واضحة** بدلاً من نص طويل:
  - Webhook URL
  - Webhook Secret (محفوظ ✓ / غير محفوظ ✗)
  - Store ID
  - الحالة (مضبوط ✓ / غير مضبوط ✗)
  - إجمالي الطلبات
  - آخر طلب
- [ ] **أيقونات للمعلومات:** أيقونة لون أخضر للحالة الجيدة، أصفر/أحمر للتحذيرات
- [ ] **تحذير منفصل:** عرض تحذير `data.warning` في banner مميز (أزرق أو أصفر) إذا وُجد

### 3. إضافة أزرار تفاعلية

- [ ] **زر "نسخ Webhook URL"** لتسهيل نسخ الرابط للوحة تحكم Easy Orders
- [ ] **زر "نسخ Store ID"** (اختياري) إذا كان مفيداً
- [ ] **زر "إغلاق" أو "تم"** بدلاً من زر "OK" الافتراضي
- [ ] **رابط سريع** إلى "إضافة/تحديث Webhook Secret" إذا كان غير محفوظ

### 4. تحسين الوصولية (Accessibility)

- [ ] **إغلاق بالضغط على Escape**
- [ ] **إغلاق بالضغط خارج النافذة** (اختياري)
- [ ] **تركيز تلقائي على زر الإغلاق** عند فتح النافذة
- [ ] **دعم قارئ الشاشة** (ARIA labels)

### 5. تناسق مع التطبيق

- [ ] **الوضع الداكن/الفاتح:** دعم `dark:` في التصميم
- [ ] **اللغة والاتجاه:** دعم RTL والخط العربي
- [ ] **الاستجابة:** أن يعمل بشكل جيد على الشاشات الصغيرة (mobile)

---

## الملفات المتأثرة

| الملف | التعديل المقترح |
|-------|------------------|
| `app/dashboard/integrations/page.tsx` | استبدال `alert(message)` بفتح Modal مع البيانات |
| مكون جديد (مثلاً `components/ui/WebhookInfoModal.tsx`) | Modal لعرض معلومات Webhook بتصميم محسّن |
| أو `components/ui/ConfirmationModal.tsx` | توسيع `customContent` ليدعم عرض معلومات Webhook بشكل منظم إن أمكن |

## هيكل المقترح للمكون الجديد

```
WebhookInfoModal
├── عنوان: "معلومات Webhook"
├── قائمة معلومات (key-value)
│   ├── Webhook URL (مع زر نسخ)
│   ├── Webhook Secret
│   ├── Store ID
│   ├── الحالة
│   ├── إجمالي الطلبات
│   └── آخر طلب
├── تحذير (إن وُجد)
└── زر إغلاق / نسخ الرابط
```

---

## أولوية التنفيذ

1. **أولوية عالية:** استبدال `alert()` بمكون Modal يتوافق مع ثيم التطبيق
2. **أولوية متوسطة:** زر نسخ Webhook URL
3. **أولوية منخفضة:** أزرار إضافية، تحسينات الوصولية

---

**آخر تحديث:** حسب طلب المستخدم — تحسين عرض معلومات Webhook في صفحة التكاملات.
